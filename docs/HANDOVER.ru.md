# Передача графа (Handover)

Одноразовый зашифрованный перенос графа Neo4j проекта между машинами. Используется при передаче проекта другому разработчику или переезде на новую рабочую станцию. Экспортирует полный граф, сжимает и шифрует его, коммитит артефакт в git, затем воспроизводит его на принимающей машине.

Не подходит для одновременной работы нескольких разработчиков с общим графом — это запланированная возможность будущего релиза.

## Предварительные требования

Оба компьютера должны иметь одинаковый набор инструментов.

| Инструмент | Назначение | macOS | Debian/Ubuntu | Arch |
|------------|-----------|-------|---------------|------|
| `age` | Симметричное шифрование | `brew install age` | `apt install age` | `pacman -S age` |
| `expect` | Неинтерактивная подстановка пароля | встроен в `/usr/bin/expect` | `apt install expect` | `pacman -S expect` |
| `gzip` | Сжатие | обычно уже есть | обычно уже есть | обычно уже есть |
| `jq` | Работа с JSON-манифестом | `brew install jq` | `apt install jq` | `pacman -S jq` |
| `docker` | Выполнение команд в контейнере | [Docker Desktop](https://docs.docker.com/get-docker/) | [Docker Engine](https://docs.docker.com/engine/install/) | `pacman -S docker` |

**Передача пароля.** Пароль никогда не коммитится в git. Передавайте его через сторонний канал: 1Password, Bitwarden или любой защищённый мессенджер, согласованный с принимающей стороной.

## Экспорт (Разработчик 1)

### 1. Убедиться, что контейнер запущен

```bash
docker ps | grep neo4j
```

Если контейнера нет — запустите стек:

```bash
docker compose -f graph-infra/docker-compose.yml up -d
```

### 2. Указать контейнер (только при нескольких проектах)

Порядок определения контейнера:
1. Переменная окружения `NACL_CONTAINER`
2. Флаг `--container=NAME`
3. `CONTAINER_PREFIX` из `graph-infra/.env` → `${CONTAINER_PREFIX}-neo4j`
4. Единственный запущенный контейнер `*-neo4j` (автоопределение)
5. Ошибка со списком кандидатов

Если запущено несколько `*-neo4j`-контейнеров, укажите нужный явно:

```bash
export NACL_CONTAINER=myproject-neo4j
```

### 3. Запустить экспорт

Минимальный вариант (пароль запрашивается интерактивно, файлы сохраняются в `graph-infra/handover/`):

```bash
./graph-infra/scripts/handover-export.sh
```

С паролем через переменную окружения и явным указанием контейнера (неинтерактивный режим):

```bash
export NACL_CONTAINER=learn-neo4j
export NACL_HANDOVER_PASSPHRASE=correct-horse-battery-staple

./graph-infra/scripts/handover-export.sh
```

Полный список флагов:

```
./graph-infra/scripts/handover-export.sh \
  [--container=NAME]          # переопределить определение контейнера
  [--to=git|s3://...]         # по умолчанию: git  (s3 пока недоступен)
  [--out-dir=PATH]            # по умолчанию: graph-infra/handover
  [--passphrase-env=VAR]      # по умолчанию: NACL_HANDOVER_PASSPHRASE
```

### 4. Что появляется в `graph-infra/handover/`

Два файла на каждый экспорт:

| Файл | Содержимое |
|------|-----------|
| `YYYY-MM-DDTHH-MM_<sha>.cypher.gz.age` | Зашифрованный сжатый дамп Cypher |
| `YYYY-MM-DDTHH-MM_<sha>.cypher.manifest.json` | Количество узлов/связей, гистограмма меток, версии — открытый текст |

Манифест не содержит чувствительных данных. Артефакт зашифрован и помечен как бинарный в `.gitattributes`.

### 5. Закоммитить и запушить

```bash
git add graph-infra/handover/
git commit -m "chore: graph handover snapshot $(date -u +%Y-%m-%d)"
git push
```

### 6. Передать пароль через сторонний канал

Отправьте через 1Password, Bitwarden или другой согласованный защищённый канал. Не добавляйте пароль в сообщение коммита, описание PR или любой файл.

## Импорт (Разработчик 2)

### 1. Получить изменения из git

```bash
git pull
```

### 2. Запустить локальный контейнер Neo4j

Импорт полностью удаляет существующие данные перед воспроизведением. Пустой контейнер подходит.

```bash
docker compose -f graph-infra/docker-compose.yml up -d
```

Подождите ~30 секунд, пока Neo4j будет готов к работе.

### 3. Запустить импорт

```bash
./graph-infra/scripts/handover-import.sh \
  --file=graph-infra/handover/2026-04-19T14-30_abc1234.cypher.gz.age
```

Полный список флагов:

```
./graph-infra/scripts/handover-import.sh \
  --file=PATH                 # обязательно: путь к артефакту .cypher.gz.age
  [--container=NAME]          # переопределить определение контейнера
  [--from=git|s3://...]       # по умолчанию: git  (s3 пока недоступен)
  [--passphrase-env=VAR]      # по умолчанию: NACL_HANDOVER_PASSPHRASE
  [--force]                   # пропустить запрос подтверждения удаления данных
```

Скрипт выполнит:
- Предупреждение об удалении всех данных в контейнере и запрос подтверждения (пропускается с `--force`)
- Запрос пароля, если `NACL_HANDOVER_PASSPHRASE` не задан
- Расшифровку и распаковку артефакта во временную директорию
- Копирование Cypher-файла в контейнер, удаление графа, сброс всех ограничений и индексов, воспроизведение Cypher
- Автоматическую проверку количества узлов и связей по манифесту

### 4. Проверить результат

```bash
# NaCl скиллы
/nacl-tl-status
/nacl-sa-validate
```

Оба должны показать ожидаемые счётчики и пройти валидацию.

## Верификация

Манифест фиксирует: количество узлов, количество связей, гистограмму меток, гистограмму типов связей, количество ограничений, версию Neo4j, версию APOC, git SHA и имя источника-контейнера. После воспроизведения скрипт автоматически сравнивает счётчики с манифестом и завершается с ошибкой при расхождении.

Для ручной проверки:

```cypher
MATCH (n) RETURN count(n) AS nodes;
MATCH ()-[r]->() RETURN count(r) AS rels;
```

Сравните значения с содержимым соседнего файла `.manifest.json`.

## Устранение неполадок

| Симптом | Решение |
|---------|---------|
| `'age' not found` | Установите `age` — см. таблицу предварительных требований |
| `'expect' not found` | Установите `expect` — на macOS он находится в `/usr/bin/expect`; на Linux: `apt install expect` |
| Ввод пароля зависает или завершается с ошибкой | Убедитесь, что `expect` установлен и `age` доступен в `PATH` |
| `Cannot detect Neo4j container` | Укажите `NACL_CONTAINER=<имя-контейнера>` или передайте `--container=NAME` |
| Автоопределение выбирает не тот контейнер | То же — укажите `NACL_CONTAINER` явно |
| Предупреждение о несовпадении версий Neo4j | Допустимо в рамках одного major.minor; при расхождении major — обновите Neo4j на целевой машине |
| Несовпадение счётчиков после импорта | Перезапустите с `bash -x` для подробного вывода; сверьте версию APOC на источнике и цели (`apoc.version()`) |
| Коммит слишком большой для git | Смотрите политику очистки в `graph-infra/handover/README.md` — храните по одному артефакту на событие передачи |

## Хранение артефактов

### Git (текущий режим по умолчанию, `--to=git`)

Артефакты сохраняются в `graph-infra/handover/`. Файл `.gitattributes` в этой директории помечает `*.cypher.gz.age` как бинарный, чтобы git не применял текстовые диффы и нормализацию переносов строк. Политика очистки и команда очистки истории описаны в `graph-infra/handover/README.md`.

### S3 (`--to=s3://…`)

Недоступно в этом релизе. Скрипты отклоняют флаги `--to=s3://…` и `--from=s3://…` с пояснительным сообщением. Поддержка S3/MinIO запланирована в будущем релизе вместе с общей инфраструктурой.

## Ограничения

- **Одноразовый перенос.** Не живая синхронизация. Принимающая машина получает снимок на момент экспорта; изменения на любой из сторон после экспорта не объединяются.
- **Предположение об одном писателе.** Не запускайте NaCl-скиллы на исходной машине во время импорта на целевой. Импорт удаляет граф перед воспроизведением.
- **Единая аутентификация.** Все машины используют одинаковый пароль Neo4j по умолчанию (`neo4j_graph_dev`, если не переопределён в `graph-infra/.env`).
- **Экспорт пустого графа отклоняется.** Если в исходном контейнере ноль узлов, `handover-export.sh` завершается ошибкой `Export produced empty file — APOC export may have failed.` Эта проверка существует, чтобы поймать отсутствие или поломку плагина APOC, но она же срабатывает на реально пустом графе. Создайте хотя бы один узел (или используйте другой механизм передачи), если нужен снимок пустого состояния.
