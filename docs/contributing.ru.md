# Участие в проекте

Спасибо за интерес к NaCl! Здесь описано, как контрибьютить скиллы, исправления и улучшения.

## С чего начать

1. Форкните репозиторий
2. Клонируйте форк и настройте скиллы ([Быстрый старт](quickstart.ru.md))
3. Создайте ветку: `git checkout -b feat/my-skill`
4. Внесите изменения
5. Протестируйте локально в Claude Code
6. Отправьте pull request

## Формат файла скилла

Каждый скилл — директория с файлом `SKILL.md`:

```
my-skill/
└── SKILL.md
```

### YAML Frontmatter (обязательно)

Каждый SKILL.md должен начинаться с frontmatter:

```yaml
---
name: my-skill
description: |
  Описание в один абзац — что делает скилл.
  Use when: фразы-триггеры для маршрутизации.
---
```

Поле `name` должно совпадать с именем директории.

### Тело скилла

После frontmatter:

1. **Декларация роли** — секция `## Your Role`
2. **Вызов** — как пользователь запускает скилл
3. **Workflow** — пронумерованные фазы с входами/выходами
4. **Параметры/Модификаторы** — все варианты вызова в таблице (см. [Конвенции модификаторов](skill-modifiers.md#conventions))
5. **Ссылки** — на общие ресурсы

## Конвенции именования

Все NaCl-скиллы следуют паттерну `nacl-{слой}-{действие}` с дефисами в качестве разделителей.

| Префикс | Расшифровка | Слой | Пример |
|---------|------------|------|--------|
| `nacl-ba-` | **B**usiness **A**nalysis (бизнес-анализ) | Анализ бизнес-процессов, сущностей, ролей, правил | `nacl-ba-context` |
| `nacl-sa-` | **S**ystem **A**nalysis (системный анализ) | Техническая спецификация: архитектура, доменная модель, UC, UI | `nacl-sa-domain` |
| `nacl-tl-` | **T**eam **L**ead (тимлид) | Жизненный цикл разработки: TDD, ревью, QA, деплой, релиз | `nacl-tl-dev-be` |
| `nacl-` | *(утилиты)* | Общая инфраструктура: core-ссылки, рендеринг, публикация | `nacl-render` |

## Языковые конвенции

- **BA/SA скиллы** (`nacl-ba-*`, `nacl-sa-*`): на русском. Язык скилла определяет язык работы Claude с пользователем.
- **TL скиллы** (`nacl-tl-*`): на английском.
- **CLAUDE.md файлы**: всегда на английском.
- **Документация (docs/)**: английский основной, русский как `*.ru.md`.

## Тестирование скилла

Перед отправкой PR:

1. Подключите скилл: `ln -sf /path/to/my-skill ~/.claude/skills/my-skill`
2. Откройте Claude Code в тестовом проекте
3. Запустите скилл и проверьте результат
4. Проверьте обработку крайних случаев (нет config, пустой граф и т.д.)

## Чеклист Pull Request

- [ ] SKILL.md имеет YAML frontmatter с полем `name`
- [ ] Имя директории совпадает с `name`
- [ ] Протестировано локально в Claude Code
- [ ] Нет захардкоженных путей (`~/projects/`, `/Users/`)
- [ ] Нет credentials или API-ключей
- [ ] Следует конвенциям именования
- [ ] Модификаторы задокументированы (если есть) по [конвенциям](skill-modifiers.md#conventions)
- [ ] Документация обновлена при необходимости

## Стиль кода (CLI-инструменты)

Для изменений в `docmost-sync/` или `yougile-setup/`:

- TypeScript с ESM-модулями
- Билд должен проходить: `npm ci && npm run build`
- Новые зависимости — только с обоснованием

## Сообщения об ошибках

Используйте [GitHub Issues](https://github.com/ITSalt/NaCl/issues) с подходящим шаблоном:

- **Bug Report** — что-то не работает
- **Feature Request** — предложение улучшения
- **New Skill Proposal** — предложение нового скилла
