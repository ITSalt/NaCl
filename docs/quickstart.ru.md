# Быстрый старт

Выберите ветку, соответствующую вашему приложению. Для Codex нормальный путь начинается в UI; для Claude сохраняются текущие каналы 2.24.0.

<!-- doc-key: prerequisites -->
## Предварительные требования

- Аутентифицированный Codex Desktop или Claude Code.
- Docker Desktop для локального графа Neo4j или доступ к VPS, где уже размещён контейнер Neo4j Community для проекта.
- Папка проекта, которую агентская среда может читать и изменять.

Для разработки из репозитория дополнительно нужны Git 2.30+ и Node.js 18+, но обычному пользователю Codex-плагина для установки не нужны ни checkout, ни терминал.

<!-- doc-key: choose-channel -->
## Выбор канала установки

Для **Codex Desktop** установите полный плагин NaCl из доверенной карточки в **Plugins**. Полностью перезапустите Codex и откройте новую задачу. Сейчас проверен локальный candidate; публичная карточка, Streamable HTTP MCP, OAuth и релиз имеют статус `NOT_RUN`. См. [установку Codex-плагина](setup/install-codex-plugin.ru.md).

Для **Claude Code Desktop или CLI** используйте [инструкцию по установке](setup/install-skills.ru.md). Сохраните GUI-проход 2.24.0 и выберите только один Claude-канал на машину.

<!-- doc-key: verify-installation -->
## Проверка установки

В новой задаче Codex отправьте:

```text
Вызови nacl_installation_doctor ровно один раз без аргументов. Сообщи status, mode, pluginVersion и executionLocation. Продолжай, только если status=VERIFIED и mode=plugin-only.
```

Версия должна совпадать с установленной карточкой, а `executionLocation` должен быть `installed-cache`. При любом отличии остановитесь. Пользователи Claude могут выполнить dry-run команду из их канала.

<!-- doc-key: run-first-dry-run -->
## Первый dry run

Попросите установленный NaCl-плагин:

```text
Используй nacl-init для этого проекта в режиме dry run. Покажи определённую идентичность проекта, режим графа, планируемые файлы, порты, маршрут к серверу и blockers. Ничего не изменяй.
```

Проверьте план. Локальный режим создаёт один контейнер Neo4j 5 Community и постоянные volumes для этого проекта. Удалённый режим подключается к контейнеру проекта на доступном VPS. Доступ к серверу сейчас подразумевает доступ ко всем базам проектов на нём; `project_scope` — маршрутизация и provenance, а не авторизация.

<!-- doc-key: initialize-project -->
## Инициализация первого проекта

Если dry run корректен, попросите `nacl-init` инициализировать проект и подтвердите только точный confirmation из текущего плана. Инициализация создаёт или подключает граф, ждёт health, загружает схему и записывает подключение без публикации сырого пароля в файлах проекта.

Начните с `nacl-ba`, продолжите `nacl-sa` и используйте `nacl-tl` для планирования и поставки. Перед закрытием запустите `nacl-verify`. Это публичные дирижёры: они сами выбирают внутренние листовые скиллы.
