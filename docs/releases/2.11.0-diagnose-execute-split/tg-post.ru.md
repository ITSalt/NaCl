NaCl 2.11.0 — diagnose-execute-split

`/nacl-tl-fix` был монолитом целиком на Sonnet — потому что заканчивается генерацией кода и целиком маршрутизировался в агента developer. Но его первые пять шагов — не генерация кода: граф-обход импакта, gap-check, классификация L0/L1/L2/L3, определение корректного поведения, авторство спеки. Это работа глубокого рассуждения, и запуск её на Sonnet ослаблял главный guardrail скилла — классификацию L3-feature.

Что вошло:

— Новый 7-й агент, `diagnostician` (Opus, high effort). Выполняет диагностико-спековую фазу fix (Шаги 1–5) как саб-агент и возвращает структурированный fix-plan. Пишет спеки/доки/узлы графа — никогда прод-код, никогда не коммитит.

— `/nacl-tl-fix` теперь двухфазный оркестратор: Фаза A (диагностика и спека) на Opus через diagnostician; USER GATE между фазами; Фаза B (исполнение: baseline → RED-first регресс-тест → правка → six-status → impact-survey) inline на Sonnet. Шов после Шага 5 — Sonnet получает готовую спеку, что восстанавливает посылку developer-слоя «кодогенерация из полной спеки».

— Фаза B — это то же honest-exec-ядро, в которое dev-be/fe --continue, reopened и hotfix уже делегируют: они тонкие обёртки над ним, а не пиры. Новая секция `## Contract` делает эту зависимость явной.

— Исправлен L1-гейт spec-first. Гейт требовал spec-update коммит для «L1 и выше», но чистый L1 не меняет доки и не может его произвести — поэтому каждый честный L1 был вынужден либо BLOCK, либо подписанное исключение. L1 теперь проходит по gap-check attestation (вердикт диагноста «доки актуальны, дрейфа нет», записанный в .tl/status.json до кода). Анти-геймнг сохранён: нет attestation → FAIL (тот самый «прыжок в код»).

— Уточнён Принцип проектирования 1: «думатели не пишут» → «думатели не пишут код». Opus-агент может писать спеки, но не код, и не коммитит. Firewall — автор спеки ≠ автор кода.

Без ломающих изменений — выходной контракт и поведение по умолчанию сохранены; Codex-вариант остаётся монолитным (в его рантайме нет саб-агентов).
