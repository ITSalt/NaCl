NaCl 2.14.0 — current-branch-batch-mode

`/nacl-goal intake` проектировался под стерильное рабочее дерево: отказ при любых незакоммиченных изменениях, свежая goal-ветка, push после каждого атома. Реальные деревья выглядят иначе — у вас одна открытая фича-ветка, в которую копится несколько фиксов перед выкаткой, а CI стоит ~5 минут за push. Хуже того: в одном checkout могут одновременно работать два-три агента, и у каждого незакоммиченные файлы. Старые прекчеки читали всё это как «грязно, отказ».

Что вошло:

— **Режим текущей ветки — новый дефолт.** С непродакшеновой ветки прогон выполняется ПРЯМО в ней: атомы коммитятся локально, ОДИН push и ОДИН прогон CI — на доставке. Прежние коммиты ветки аннотируются в теле PR; goal-коммиты несут трейлер `Goal-run-id:`, так что `git log --grep` (и `git bisect`) отделяют их начисто. `--branch=new` возвращает старый поток байт-в-байт; `main`/`release/*` отказывают как и раньше.

— **Smart WIP.** Незакоммиченные файлы считаются работой другого агента — не стейджатся, не откатываются и больше не повод для отказа. Защита от пересечений двухслойная: предсказание зоны правок до старта (один консолидированный вопрос — только при реальном пересечении) и жёсткий гейт в момент коммита — ship стейджит селективно (никогда `git add -A` в общем дереве) и отказывается трогать пути из снапшота; коллизия даёт единственный *resumable* GOAL_BLOCKED-код. Регрессионные baseline/postfix гоняются в одноразовых worktree на зафиксированных SHA — живой WIP соседа не загрязнит дифф.

— **Автономный оркестратор перестаёт спрашивать «я нашёл ошибку — подтверждаете, что это ошибка?»** Под флагом `--autonomous` (его передаёт только обёртка) launch-sanity подтверждается сам — вызов оркестратора И ЕСТЬ намерение; атомы средней уверенности роутятся по ведущей гипотезе с записанной и раскрытой альтернативой; атомы низкой уверенности собираются в ОДИН вопрос до старта. Биллинг, auth, миграции схемы, деструктивные операции, продуктовые решения — по-прежнему отказ до `/goal`, всегда. Автономия расширяет роутинг, но не глотает критические вопросы.

Обратная совместимость сквозная: старые артефакты прогонов валидны, отсутствие новых env vars означает поведение pre-2.14, в GOAL_PROOF ключи только дополняются advisory-хвостом.
