NaCl 2.12.2 — ship-next-step-names-skill

Финальный блок `Next step:` у `nacl-tl-ship` обязан всегда называть конкретную команду `/nacl-...`, а не прозу. Ship — это передача эстафеты, и она работает только если оператору названа точная следующая команда. Патч закрывает путь, где ship выдавал прозой «review and merge the PR» вместо имени скилла, и вшивает merge-скилл в подсказку, чтобы «теперь нужен merge» резолвилось в исполняемую команду.

Что вошло:

— Дыра: bare bug-fix ship (статус уже PASS, без UC-id, PR открыт в нестандартную базовую ветку) заканчивался строкой `Next step: Review and merge the PR` — без имени скилла. Корень: шаблоны ship предполагали UC-flow и наличие UC-id, и не было варианта для «PASS + открытый PR + нет id», поэтому модель заполняла строку прозой. Merge не назывался, потому что он ниже по течению от ship — `/nacl-tl-release` единственный скилл, который выполняет `gh pr merge`.

— Новый инвариант: блок `Next step:` теперь содержит только конкретные команды `/nacl-...` (с реальными флагами), без прозы; при отсутствии UC/FR-id аргумент опускается, а не подменяется описанием.

— Таблица резолюции: ship заполняет блок по данным, которые у него уже есть (strategy, номер PR + резолвнутая базовая ветка, статус верификации, наличие id). Для bare-fix она называет `/nacl-tl-release --pr <N>` как маршрут merge и прямо фиксирует: release — единственный merge-скилл, он читает базу из `git.main_branch` (никакого захардкоженного `main`) и гейтит по PASS. Готовый пример bare-bug-fix повторяет ровно тот сценарий, чтобы у модели был образец.

— Codex-паритет: сжатый вариант `skills-for-codex/nacl-tl-ship` зеркалит методологию (шаг Workflow, называющий конкретный следующий скилл), удовлетворяя root↔Codex sync-гейту.

Без ломающих изменений — блок `Next step:` это пользовательская подсказка, а не часть машинного output-контракта ship, поэтому аудит контракта у downstream-потребителей не нужен; используется только существующий флаг `--pr`, так что CI-гарды остаются зелёными. Подсказки next-step остаются информационными (без авто-чейнинга).
