NaCl 2.19.0 — skill-tools

Методичка по Agent Skills (Anthropic, июнь 2026) проводит одну черту: скрипты — для детерминированных операций, естественный язык — только для настоящих суждений. Механизм важен: когда детерминированная процедура живёт в прозе `SKILL.md`, агент её не вспоминает — он **заново выводит и исполняет её рассуждением каждый прогон**, а рассуждение не воспроизводимо между прогонами и моделями. На длинной автономной петле это капель разброса и тихих ошибок. У NaCl уже был ровно один такой тул (`classify-status.mjs`); 2.19.0 обобщает паттерн — и выходит вместе с измерением.

Что внутри:

— **Пять выносов.** branch-slug + base-branch гард, раскладка по волнам, rollup severity валидации, CI-watch + health-проба, формат BA-id — стали single-authority скриптами, запинёнными тестами, сведёнными в `nacl-core/` и прошитыми в **каждый** скилл, который это решение принимает. Каждый вынос поведенчески-нейтрален.

— **Измерение, а не вера.** A/B-харнесс прогнал то же решение двумя руками — OLD (правила прозой, модель считает руками) vs NEW (запусти тул) — `{Haiku 4.5, Opus 4.8} × 3 решения × N=20` = **240 вызовов `claude -p`** (пейсинг через `itsalt-pinch`). NEW: 6/6 ячеек `1 distinct · 100%`, и модели реально делегировали. OLD деградирует по типу логики: раскладка по волнам **5% (Haiku) / 20% (Opus)** руками; rollup-валидация **40% на Opus** на границе. Контринтуитив: сильная модель руками **хуже** — нельзя предсказать, что тихо сломается. Это и снимает тул. ($16, запустить: `node bench/skill-tools/ab/ab.mjs`.)

— **Перепроверено вживую.** На реальном `/nacl-sa-validate` по `family-cinema` тул реально вызвался (видно в транскрипте), перезапуск на том же входе дал байт-в-байт тот же вердикт и совпал с ручным оракулом. С дословным текстом скилла на граничном кейсе слабая модель перевернула гейт `PASS→WARN` 1 раз из 6; тул — нет.

— **Та же ошибка была по всему фреймворку.** Живой `--feature` план вскрыл: `wave-plan.mjs` обходился на инкрементальном пути (grep по 127 транскриптам — тул не вызывался ни разу). Аудит нашёл тот же паттерн: deploy/deliver переписывали `gh run watch` + health-`curl` руками; ba-validate — rollup; hotfix/conductor — slug; а `ba-sync` форматировал id с усечением (`GPR-00` на сотой группе) против канонического `apoc.text.lpad` у модельных скиллов. Всё сведено в `nacl-core`, прошито, расхождение id закрыто каноном.

— **Возьмите на свой проект.** `VERIFICATION-PLAYBOOK.md` — протокол, как самому доказать, что вынесенный тул точнее и лучше прозы (независимый оракул, edge-кейсы, правило победы V1∧V2∧V3, downstream-проверка).

84 тул-теста зелёные, CI-гейты чисты, root/Codex sync VERIFIED, приватность-канарейка чиста. Честно: `nacl-ids` — самый лёгкий вынос (формат id естественнее в Cypher), модельные скиллы не переписывал; живой Tier B end-to-end не делался (auth не переживает CLAUDE_CONFIG_DIR).
