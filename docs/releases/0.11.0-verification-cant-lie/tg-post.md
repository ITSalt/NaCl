NaCl 0.11.0 — Verification Skills Cannot Lie

Wave 2 of the "honest skills" series: after 0.10.0 fixed the bug-fix skill and 0.10.1 propagated the fix to downstream consumers, the verification pipeline itself was still returning PASS without running tests. Five skills updated in one wave. nacl-tl-verify-code now discovers scripts.test, runs it, and checks test coverage of the changed file — static analysis alone produces UNVERIFIED, not PASS. nacl-tl-stubs adds a test-file scan pass that catches hollow describe() blocks with zero it()/test() calls (the "44-stub scenario" the old comment-marker scan missed). nacl-tl-sync runs both BE and FE test suites after static checks and verifies endpoint path coverage; nacl-tl-review flags MAJOR when tests and production code share the same author (>50% git log overlap). All five skills gain a ## Contract section with the downstream-consumer audit discipline.
