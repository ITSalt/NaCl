NaCl 2.23.0 — multi-user-shared-graph

Несколько программистов — один граф. Спецификационный граф проекта теперь может жить на VPS и быть общим для нескольких разработчиков через интернет. Локальные проекты не тронуты вообще: нет `graph.mode` ⇒ всё как раньше, байт в байт.

Раньше каждый проект держал свой локальный контейнер Neo4j (`bolt://localhost:<port>`), по одному на машину, а совместная работа сводилась к разовому зашифрованному хэндоверу — снимок, а не живой общий граф.

Как это устроено:

— Скиллы общаются с графом только через MCP-сервер `neo4j`, поэтому точка абстракции уже была в нужном месте. В режиме `graph.mode: remote` MCP по-прежнему смотрит на `bolt://localhost:<sidecar_port>`, а персональный mTLS-туннель (ghostunnel) уводит этот локальный сокет на гейтвей перед Neo4j на VPS. Итог — ноль изменений в скиллах ради связности.

— Доступ = персональный клиентский сертификат, отзываемый «API-ключ», подписанный приватным CA (без Let's Encrypt). Neo4j наружу не смотрит — открыт только mTLS-гейтвей. Редакция — Neo4j Community, без лицензионных затрат.

Что внутри:

— Опциональный конфиг с полной обратной совместимостью: `graph.mode` (`local` | `remote`, отсутствует ⇒ `local`).
— Семь детерминированных инструментов с юнит-тестами (паттерн skill-tools); `nacl-init` похудел — реестр и mcp-merge вынесены в тулы.
— Провижининг VPS под ключ: private CA, серверный серт, Neo4j на loopback + ghostunnel-гейтвей, загрузка схемы, firewall default-deny, выпуск/мгновенный отзыв клиентских сертов.
— Миграция local→cloud одним скриптом с откатом на каждом шаге.
— Конкурентность без коллизий: граф — единственный источник правды, claim-локи на задачах, авто-идентичность на машину (`<git email|user>/<machine-key>`), так что один человек на двух машинах никогда не берёт одну и ту же задачу.

Проверено на реальном VPS:

Зарелижено после первого живого end-to-end прогона на чистой Ubuntu: провижининг → миграция реального графа в облако → подключение второй машины → два разработчика берут задачи одновременно без коллизий → отзыв серта мгновенно лишает доступа одного, второй продолжает работать; счётчики узлов и рёбер до и после миграции совпадают байт в байт. Скрипты до этого вживую не запускались — прогон вскрыл девять фиксов, которые уже внутри. Самые показательные: у ghostunnel нет флага `--crl`, поэтому отзыв стал управляемым allow-list по CN; а современный Go-TLS отвергает серверный серт только с CN — теперь серт несёт subjectAltName. Windows в том же заходе получил паритет по сайдкару и идентичности.

Локальный режим не тронут. Открытый follow-up один: дедуп ресолвера mcp-бинарника в `setup-graph.{sh,ps1}`.
