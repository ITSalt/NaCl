NaCl 2.15.0 — connected-spec-graph

Самый большой релиз со времён появления самого графа. До 2.15 граф спецификации описывал систему, но не реагировал на изменения: рёбра воздействия, которые никто не читал; task-файлы с запечёнными снапшотами, тихо устаревавшими; никакого понятия «этот узел надо пересмотреть, потому что изменился его upstream»; обоснования решений — только в сообщениях коммитов. Добавлять новые типы артефактов поверх такой модели изменений значило усиливать дрейф. Поэтому релиз сначала чинит модель изменений — и только потом наслаивает четыре новых слоя спецификации.

Что внутри:

— **Распространение изменений + provenance (Фаза 0).** Потяни за любой узел — граф вычислит всё зависимое и поставит направленный, точечный `review_status` ровно на затронутую работу. Широкий обход — только для разведки: измерено, он перепомечает в 20×→52× раз больше (и насыщается). Каждое решение — узел `:Decision` с цепочками JUSTIFIES/SUPERSEDES: «почему так устроено» через год — один Cypher-обход.

— **Четыре новых слоя, один инвариант (Фазы 1–4).** Машины состояний экранов (валидаторы детерминизма, достижимости, retry-гарантии), behavior slices (graph-native Given/When/Then с жёстким якорным инвариантом), таксономия доменных ошибок (транспортно-независимые коды, канал-правило обработки, презентации на языке пользователя), политики кэша и деградации («когда кэш перестаёт врать»; состояние, в котором пользователь проживает отказ). Каждый новый тип узла рождается с обязательным родителем, обязательным cross-layer якорем, регистрацией в allow-list обхода и собственным уровнем валидатора — осиротеть структурно нельзя.

— **Доказательства, не обещания.** Каждая фаза: бенчмарк на изолированном клоне графа реального проекта с фальсифицируемыми гипотезами, матрицы инъекции дефектов 8/8 → 14/14 → 21/21 → 27/27 с нулевым cross-talk, затем независимый skill-прогон со слепым верификатором. Готовый роадмап прошёл внешний экспертный аудит — а сам аудит проверен попунктно, с live-replay всех четырёх харнессов байт-в-байт против эталонных результатов. Пять лабораторных отчётов + серия из пяти статей в репозитории.

— **Путь апгрейда для существующих графов.** `docs/upgrade-graph-extensions.md`: отдайте агенту с чистым контекстом — он оркестрирует весь апгрейд через сабагентов, задаёт ОДИН вопрос на старте (источник правды: код или вы?), дальше спрашивает только про бизнес-поведение и заканчивает сверкой кода со спецификацией.

Всё аддитивно и строго opt-in — вакуумный проход подтверждён бенчмарком: графы, не принявшие слой, его валидаторами не затрагиваются.
