**NaCl 0.15.0 released — dev & verification contract cleanup**

0.13.0 made every leaf skill produce honest six-status output. 0.14.0 made the orchestration paths to `main` / deploy / delivery / release honour it. 0.15.0 closes the layer in between — the dev `--continue` paths, the verifier and sync baselines, the review runner discovery, and the wave / intake / deploy stamping. Every implementation and quality gate now consumes or emits the same six-status contract with exact runner discovery and explicit baseline evidence.

Highlights:
— `nacl-tl-dev`, `nacl-tl-dev-be`, `nacl-tl-dev-fe`: `--continue` no longer self-grades. The inline "fix the issue, run tests" loop is replaced with a delegation to `/nacl-tl-fix "<problem>" --uc UC### --from-review`. The fix sub-agent owns runner discovery, baseline capture, RED-first regression test via `/nacl-tl-regression-test`, postfix run + set-difference, and six-status determination. The dev skill parses `Status:` from the fix report, verifies the regression-test seam (`Tests > Regression test` path + `Tests > RED→GREEN` evidence — silence is `UNVERIFIED`), and promotes the phase to `ready_for_review` only on `Status: PASS` (or operator-gated accepted-`BLOCKED`). The dev skill never writes test files in `--continue`.
— `nacl-tl-fix`: new `--from-review` flag (metadata-only). Marks the invocation source as `review` in the report and the changelog. The six-status contract, baseline procedure, and RED-first discipline are unchanged.
— `nacl-tl-verify-code` and `nacl-tl-sync`: explicit baseline-ref discovery. Resolution priority is `--base <ref>` flag → saved `.tl/tasks/<id>/baseline-failures-{be,fe}.json` artifact → `git merge-base HEAD main`. The baseline runs via `git worktree add` to a temp dir; the worktree is removed on every exit path. No baseline ⇒ `UNVERIFIED (no baseline)`, not `BLOCKED` or `REGRESSION` — set arithmetic is undefined when one operand is missing. `nacl-tl-sync` also drops its self-contradictory "Both suites pass AND pre-existing failures remain → BLOCKED" rule.
— `nacl-tl-review`: declared `scripts.test` only — no `npm test` fallback. Missing infra halts as `REVIEW HALTED — NO_INFRA`; runner crash halts as `REVIEW HALTED — RUNNER_BROKEN`. `Code judgment: APPROVED` is forbidden under both halt headlines; operator override produces `REVIEW APPLIED — UNVERIFIED (no test infra)` with `Code judgment: CHANGES REQUESTED` (never `APPROVED`).
— `nacl-tl-reopened`: declared `scripts.test` / `scripts.build` only — no `npm test` / `npm run build` fallbacks. Re-run gate halts on missing infra (no longer "proceed with warning"). Batch sub-agent prompt now includes Step 7.5 status-line parsing and Step 7.5.1 regression-test-seam evidence gate. Final template's `Status: DevDone ✅` is gated on the `REOPENED COMPLETE` headline.
— `nacl-tl-full`: Wave 0 TECH flow reads `/nacl-tl-dev TECH-###`'s six-status result before advancing — mirrors BE/FE branching. `--skip-qa` sets `phase_qa = 'skipped'` (not `'pending'`) and forces `FULL APPLIED — UNVERIFIED (qa skipped)`; UCs that completed every other phase land at `verified-pending`, not `done`. Graph write failure halts phase advancement (the "continue with status.json only" path is removed).
— `nacl-tl-deploy`: `verified-pending` upstream halts as `DEPLOY HALTED — UNVERIFIED (upstream verified-pending)`; operator override emits `DEPLOY APPLIED — UNVERIFIED` and refuses to move the source Task to `done` / `released`. Unknown verification state halts unconditionally — no "warn and proceed".
— `nacl-tl-intake`: progress and final-summary rows surface the verbatim downstream `Status:` value (`PASS`, `UNVERIFIED`, `NO_INFRA`, `RUNNER_BROKEN`, `REGRESSION`). Final state movement requires PASS-family downstream status; otherwise the atom is `unfinished` with the specific status, and the headline rolls up the worst non-PASS into `INTAKE TRIAGE INCOMPLETE — REGRESSION` / `INTAKE TRIAGE HALTED — RUNNER_BROKEN` / `INTAKE TRIAGE APPLIED — UNVERIFIED`.

Five cross-cutting principles thread through the release: `Status:` is the only authoritative classifier; declared workspace commands only (no `npm` fallbacks); baseline before any "pre-existing" / "regression" claim; skip ⇒ unverified, never PASS; no autonomous branch switching.
