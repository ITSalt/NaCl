NaCl 2.10.3 — clean-isnt-complete

SA-миграция могла отрапортовать чистый аудит, хотя почти каждый use case был записан в граф пустой оболочкой. Этот релиз закрывает обе половины проблемы.

Почему так вышло:

— Аудит сравнивает ожидаемые из IR счётчики с живыми счётчиками графа, но «ожидаемое» берёт из того же IR, который только что записал. То есть он доказывает «что распарсили — то и записали», но никогда «что распарсили — полно». Он был структурно слеп к недо-извлечению.
— Парсер недо-извлекал на нескольких распространённых диалектах документов, поэтому use case'ы попадали в граф с пустым модулем, нулём шагов сценария и нулём связей с формами.

Что вошло:

— Исправления парсера inline-table SA-адаптера: 4-значные id use case'ов (`UC-NNNN`), fallback на модули из заголовков `## 1. …` (модули теперь заполняются), вывод связей экран→use case из frontmatter (рёбра `USES_FORM` теперь заполняются) и fallback на нумерованный список шагов под «основным сценарием» (шаги активности теперь заполняются).
— Новое измерение полноты/coverage в `validate_sa_ir.py` (SC1–SC7): доли «заполнено/всего» по каждому типу узлов с примерами недостающих id. Недо-извлечение видно как, например, `UC activity steps: 1/50 (2.0%)` вместо тихого «clean».
— Coverage по умолчанию совещательный (часть пустоты легитимна); `--strict` / `--min-coverage` включают его как гейт для CI.
— `audit_sa.py` теперь документирует слепое пятно сверки счётчиков и печатает указатель на секцию coverage, чтобы чистый аудит не путали с полной миграцией.

Без ломающих изменений — coverage аддитивен, коды возврата по умолчанию не меняются.
