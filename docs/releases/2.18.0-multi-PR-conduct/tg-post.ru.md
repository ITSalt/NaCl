NaCl 2.18.0 — multi-PR-conduct

`/nacl-goal` должен уметь доводить до конца *гетерогенную* цель — несколько несвязанных правок по разным модулям — так, как это сделал бы тимлид: разложить работу по кучкам, упорядочить так, чтобы зависимости приземлялись первыми, шипить каждую кучку отдельным ревьюабельным PR и итерировать каждую, пока не станет зелёной. У NaCl эта машина уже была (`nacl-tl-conductor`: волны, lifecycle на задачу, до 3 ретраев) — просто до неё нельзя было дотянуться через `/nacl-goal`. Единственным автономным оркестратором был `intake`, унитарный по дизайну — одно намерение, одна ветка, один PR — и он *отказывается* от цели, которую пришлось бы дробить по модулям. Этот отказ был указателем на отложенную фичу из роадмапа. 2.18.0 её выпускает: новый алиас-сиблинг `conduct`.

Что внутри:

— **Одно правило выбора.** Одна связная правка одной области → `intake`. Несколько несвязанных правок по разным модулям → `conduct`. Алиасы отказывают *друг в друга* (`intake`→`conduct` на гетерогенной цели; `conduct`→`intake` на гомогенной), так что ни один молча не делает работу другого. `intake` не изменён ни на байт — multi-PR никогда не дефолт, только именованный алиас.

— **Кластеры и волны.** `conduct` разбивает классифицированные атомы на module-aligned кластеры (точная инверсия split-детектора `intake`), строит DAG кластеров и шипит каждый кластер своей веткой — отрезанной от общей `integration/goal-<hash>` — отдельным PR, в волновом порядке по межкластерным зависимостям. Graph-less по дизайну: берёт семантику `nacl-tl-conductor` без зависимости от Neo4j, прогоняя существующие inner-скилы по разу на кластер — поэтому работает на проектах без SA-графа.

— **Итерируй-до-зелёного, но bounded.** Когда у кластера UI-приёмка, `conduct` крутит петлю `/nacl-tl-qa` → фикс → ретест для CRITICAL / MAJOR-багов, не более 3 итераций; MINOR откладываются. «Столько итераций, сколько нужно» без потолка — это как раз способ, которым модель замусоливает тест до зелёного; потолок плюс stop-and-ask сделаны намеренно.

— **Частичная поставка — честное состояние.** Падение кластера не роняет соседей: зависимые пропускаются, независимые продолжают, а слитая смешанная волна приземляется в `GOAL_BLOCKED_PARTIAL_WAVE` — выборочно резюмируемое через `/nacl-goal resume --clusters=<ids>`, что перезапускает только заблокированные кластеры и не трогает уже зелёные PR. `GOAL_OK` недостижим, пока хоть один кластер заблокирован — ничего не теряется молча.

Выходит как v1: контракт + check-скрипт + интеграция inner-скилов, покрыто фикстура-тестами и чисто по CI-гейтам. v1 последовательный; живой multi-cluster acceptance-прогон, параллельные кластеры и запись Task-статуса в граф — следующие вехи.
