NaCl 2.20.0 — type-aware-release-gate

Проект на NaCl спотыкался на каждом релизе. `/nacl-tl-release` отказывал в мерже с `RELEASE HALTED — MISSING TASK NODE` — хотя каждый фикс проходил через `/nacl-tl-fix` и `/nacl-tl-ship` ровно как задумано. Воркфлоу был правильным; ошибка категории была в гейте. Затянутый в 0.13.0 до graph-only, pre-merge-гейт гонял один запрос на *каждый* PR — `MATCH (t:Task) WHERE t.id IN [...]` — и халтил, если Task-узла нет. Но это предполагает, что каждый PR — запланированная фича. Путь багфикса осознанно записывает фикс как узел **Decision**, а не Task — поэтому корректно записанный фикс не мог пройти гейт никогда. Этот релиз делает гейт type-aware.

Что внутри:

— **Гейт ветвится по типу PR.** Фичовые PR (`feat:`) сохраняют проверку Task-узла без изменений. Фикс-PR (`fix:`) проверяются по узлу **Decision**, который `nacl-tl-fix` и так создаёт (L2/L3-spec-gap), либо по code-only маркеру `Fix-level` (L0/L1) — а не по Task-узлу. Фикс больше никогда не халтится из-за отсутствия Task, которого у него и не должно было быть.

— **Дрейф всё ещё ловится.** Гейт по-прежнему HALT'ит на настоящем незаписанном дрейфе спеки — фикс меняет поведение, но за ним нет принятого Decision. Гарантия 0.13.0 «граф — источник истины» сохранена, просто проверяется против правильного узла.

— **Детерминированная связка PR→граф.** `nacl-tl-fix` штампует трейлеры `Fix-level:` / `Fix-decision:` в коммите фикса (squash-safe, в теле коммита); гейт релиза читает их и находит Decision без догадок. Сборные PR несут несколько — проверяются все.

— **Вердикт — протестированный инструмент.** `nacl-core/scripts/classify-pr-merge.mjs` — чистый, не открывает Neo4j, запинён тестами, гоняется в CI. Вердикты `MERGE` / `USER_GATE` / `HALT` со строкой graph-proof, питающей новую колонку «Graph proof» в плане мержа.

Проверено на replay-фикстуре, собранной из реального сборного PR, который халтился: фикс с Decision теперь мержится без Task-узла; фикс, заявляющий отсутствующий Decision, по-прежнему халтится. Все тесты инструментов зелёные; codex-sync и lint-гейты чисты.
