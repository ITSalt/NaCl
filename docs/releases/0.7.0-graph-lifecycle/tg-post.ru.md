**NaCl 0.7.0: автоматическое отслеживание жизненного цикла IntakeItem**

Два скилла теперь поддерживают состояние графа автоматически. `nacl-tl-deliver` помечает IntakeItem как доставленные (`status='delivered'`, `delivered_at`, `delivered_pr`) после прохождения health check стейджинга. `nacl-tl-release` проставляет версию релиза на всех доставленных IntakeItem после того, как git-тег запушен. Оба шага работают в режиме WARN+continue — недоступность Neo4j никогда не блокирует CI/CD.

Изменение мотивировано наблюдением Wave 8 в одном из рабочих проектов: пять IntakeItem оставались в статусе `draft`, несмотря на то что их код уже был в production. Операторам приходилось делать сверку вручную. Начиная с версии 0.7.0 эта сверка выполняется автоматически.

Также в релизе: stdlib-только харнесс для бенчмаркинга скиллов, рассчитанный на воспроизводимые и публикуемые в статьях тесты производительности.

Примечания к релизу: `docs/releases/0.7.0-graph-lifecycle/release-notes.md`
