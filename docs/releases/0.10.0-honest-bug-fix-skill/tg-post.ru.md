**NaCl 0.10.0: честный скилл починки багов**

`/nacl-tl-fix` больше не врёт о покрытии тестами. До 0.10.0 скилл мог увидеть «0 тестов собрано» и отрапортовать `[✓] Unit tests pass — FIX COMPLETE`, объявив пред-существующие падения «не связанными с фиксом» без какой-либо проверки. В этом релизе: TDD-порядок шагов фикса, сравнение со снэпшотом до изменения, честная таблица статусов — `PASS` / `BLOCKED` / `UNVERIFIED` / `NO_INFRA` / `RUNNER_BROKEN` / `REGRESSION` — у каждого свой заголовок отчёта и своя рекомендация next-step.

Главная структурная перемена: регрессионный тест для бага пишется **до** применения фикса, отдельным саб-агентом (новый скилл `nacl-tl-regression-test`). Тест обязан быть RED против сломанного кода; только после этого применяется фикс; только после этого отчёт имеет право сказать `FIX COMPLETE`. Вопрос «а честный ли это тест?» снимается по построению — агент больше не оценивает собственную работу.

В этот же релиз: `/nacl-sa-validate` теперь обнаруживает schema drift в pre-flight через `db.labels()` / `db.relationshipTypes()` и HALT-ит с понятной диагностикой вместо тихого FAIL'а с 7 false-positive CRITICAL-находками на графах с не-каноничными SA-лейблами (`:SAModule`, `:SAEntity`, `TRACES_TO`). XL6.1 / XL6.4 также принимают английский `'Automated'` в дополнение к русскому `'Автоматизируется'`; L1.4 толерантна к `EnumValue.code` / `.label`, а не только `.value`.

Также в релизе — та же форма, другой слой: swimlane-диаграммы активности молча деградировали в single-lane режим. Три скоординированных фикса: SA-парсер `inline-table-v1` теперь канонизирует actor каждого шага (тянет колонку `Компонент` / `Исполнитель`, разбирает строки вида `Система (триггер: ...)`, `ACT-01 Пользователь (Посетитель)`); MERGE-шаблон `nacl-sa-uc` пишет `as.actor` вместо устаревшего `as.step_type`; `nacl-sa-validate` получает проверку **L3.5 (CRITICAL)** для пустого `ActivityStep.actor` и **L3.6 (WARNING)** для неканоничных значений. Текст предупреждения рендерера теперь `actor не задан` — в соответствии со свойством схемы. Графы больше не проходят валидацию как «здоровые», пока рендерер тихо падает в single-lane за warning-баннером.

Общая нить всех трёх изменений: quality gate, который врёт о находках, хуже отсутствия quality gate. Этот релиз превращает каждый silent-FAIL путь в громкую, действенную диагностику.
