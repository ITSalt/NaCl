NaCl 2.24.0 — desktop-plugin

NaCl теперь нативный житель Claude Code Desktop. Графовый рантайм закалён под GUI-хост, и появился второй, независимый канал установки — коммиченный плагин Claude Code.

Проблема была в том, что графовый рантайм писался CLI-first: резолв docker шёл по shell PATH, у sidecar не было автозапуска на уровне ОС, а `neo4j-mcp` запускался непинованной npm-командой. Из терминала это работало, но плохо подходило под GUI-приложение — бинарники Docker Desktop часто вне PATH GUI-процесса, нет shell-сессии, чтобы оставить контейнер жить, а непинованная зависимость — дыра в воспроизводимости. Отдельно: единственный способ запустить скиллы NaCl внутри Desktop был — симлинкануть checkout репозитория так же, как это делает CLI, а это не то, для чего сделан родной механизм расширений Desktop (плагины).

Что внутри:

— **Графовый рантайм.** `graph-doctor.mjs` — отдельная liveness-проба контейнера графа и MCP-бинарника, умеет `--fix` (запустить стек), подключена как SessionStart-хук — упавший граф ловится до запуска скилла, а не посреди него. `setup-graph` теперь ищет Docker CLI и демон по GUI-app PATH-кандидатам и сам запускает Docker Desktop, если демон доступен, но не запущен. Бинарник `neo4j-mcp` теперь запинован по версии и проверяется по sha256 (по умолчанию v1.5.3, переопределяется `NEO4J_MCP_VERSION`) вместо непинованного npm-лаунчера. У sidecar — нативный автозапуск ОС: LaunchAgent на macOS, Scheduled Task на Windows. Все 11 скиллов, которые могут упереться в упавший граф, теперь выдают одно каноническое HALT-сообщение, закреплённое lint-гейтом.

— **Плагин Claude Code.** `scripts/build-plugin.mjs` собирает `plugin/` из `scripts/plugin-manifest.json`, артефакт закоммичен. 53 из 59 скиллов — как slash-команды `/nacl:<name>`, 7 профилей агентов — как `@nacl:<name>`, `nacl-core` зашит целиком как общая библиотека, graph-infra тоже внутри. Два SessionStart-хука: проверка живости графа и предупреждение о совмещении каналов. Установка — две команды:

```
/plugin marketplace add ITSalt/NaCl
/plugin install nacl@nacl
```

MCP-сервер neo4j по-прежнему настраивается на проект через `/nacl:init`, не плагином — имена тулов не меняются.

Выберите канал:

| Вы используете | Канал | Установка |
|---|---|---|
| Claude Code CLI | Симлинки (без изменений) | `sh scripts/install-claude-code-skills.sh` |
| Claude Code Desktop | Плагин (новое) | `/plugin marketplace add ITSalt/NaCl` → `/plugin install nacl@nacl` |
| Codex | `skills-for-codex/` (без изменений) | `sh skills-for-codex/scripts/install-user-symlinks.sh` |

Оба канала Claude Code технически работают в обоих хостах, но **не ставьте оба на одну машину** — задублируете каждый скилл под двумя именами. SessionStart-хук плагина предупредит, если увидит уже установленные симлинки.

Что осознанно не вошло:

— `/nacl-goal` в Desktop — оборачивает CLI-only команду `/goal`, которую Desktop не может выполнить, поэтому исключён из манифеста плагина.
— `nacl-postmortem` — редкий/высокорисковый, панель workflow остаётся на стороне репозитория.
— три `nacl-migrate*` — редкие, нужен полный checkout репозитория и Python.
— живой чек-лист Desktop (T-1…T-7) — эта версия поставляет ручную смоук-матрицу графа Desktop (`tests/desktop/`) как проверку stage-3, но полный живой прогон через Claude Code Desktop на реальном проекте ещё не делался — открытый follow-up.
