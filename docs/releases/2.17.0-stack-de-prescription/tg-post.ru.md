NaCl 2.17.0 — stack-de-prescription

Фреймворк перестаёт решать за вас, какой Node ставить. Агент в реальном проекте зафиксировал архитектурное решение, переопределяющее «Node 22.x — дефолт шаблона nacl-init» и процесс-менеджер PM2, которые он считал требованием фреймворка. Ни то, ни другое никогда не было решением фреймворка: NaCl — это методология, а не стек. Пины утекли из шаблонов CI/CD-workflow, а формулировки «MUST follow» в справочниках TL превратили конвенции одного стека в кажущиеся мандаты. У методологии не должно быть любимого технологического стека — этот релиз убирает все технологические дефолты фреймворка.

Что внутри:

— **Версии выбирает проект.** Шаблоны deploy и docker-compose dev несут плейсхолдеры `${NODE_VERSION}` / `${POSTGRES_VERSION}` / `${REDIS_VERSION}` вместо пинов; PM2 помечен как один из примеров процесс-менеджера. Новый CI-гейт (`scripts/check-version-pins.sh`, escape hatch `# version-pin-ok`) валит любой будущий PR, возвращающий пин в project-facing шаблон. Собственная графовая инфраструктура NaCl пины сохраняет — тот стек как раз и есть решение фреймворка.

— **Stack-профили вместо молчаливого закона.** Богатые гайды по Node/TS/React (code style, frontend rules, чеклисты ревью, dev environment) сохранены целиком — но теперь открываются явной шапкой «Применимость — Stack profile: Node/TS»: они действуют, когда так говорит `config.yaml → modules.<m>.stack`, а другие экосистемы адаптируют принципы. «MUST follow» стало «SHOULD для Node/TS-проектов». React-проект не теряет ничего; Django-проекту перестают молча скармливать React-правила.

— **Config-first обнаружение раннера.** `nacl-tl-dev-be/-fe` разрешают тестовую команду в порядке: `config.yaml → modules.<m>.test_cmd`, затем нативная детекция экосистемы (Node: `package.json scripts.test`), затем NO_INFRA. Ядро честного TDD не тронуто ни на букву: запускай ровно найденную команду, никогда не выдумывай раннер. Захардкоженная таблица «FE Technology Stack» удалена.

— **nacl-init никогда не выдумывает стек.** Детекция — упорядоченный проход по манифестам восьми экосистем (Node, Python, Go, Rust, JVM, .NET, PHP, Ruby); неоднозначность — повод спросить пользователя, а не подставить встроенный дефолт. Проверено смок-тестом: init на чистой директории выдаёт `stack: "unspecified"` и ноль названных технологий.

Единственный источник истины о стеке проекта — `config.yaml → modules.*`. Не шаблон, не справочник, не дефолт фреймворка.
