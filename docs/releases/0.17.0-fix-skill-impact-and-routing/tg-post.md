**NaCl 0.17.0 released — fix skill: graph impact traversal & feature routing exit**

0.15.0 made every dev and verification path consume the same six-status contract with explicit baseline evidence. 0.17.0 closes two structural gaps upstream of that, inside `/nacl-tl-fix` itself: Step 1's graph probe was keyword UC-name matching (so multi-UC blast radius around a shared catalog / table / DomainEntity was invisible), and Step 5's "L3 (Spec-create)" path quietly turned the fix skill into a feature factory (whole missing features shipped as "fixes" with type assertions, no FeatureRequest artifact, no planning, no TDD).

Highlights:

— **Step 1 — entity-driven graph traversal.** Three Cypher stages replace the single keyword query. Stage 1 resolves the touched `DomainEntity` from the affected file / SQL table. Stage 2 enumerates every UC that reads or writes the entity via `CONSUMES | PRODUCES | MUTATES | REFERENCES | AFFECTS_ENTITY`, plus 2-hop `DEPENDENCY` neighbours. Stage 3 keeps the legacy keyword search as a secondary probe. The TRIAGE table is required to list every UC returned by Stage 2; missing the entity is a hard `IMPACT_UNVERIFIED` flag in the Step 8 report, not a silent grep fallback.

— **Fix Levels — L3 split.** L3 is replaced by `L3-spec-gap` (code path exists; one minor doc / enum value / UC node is missing — inline OK, < 1 file) and `L3-feature` (NOT a fix — STOP and route). A new classification criterion makes the escalation unambiguous: any new HTTP route, new DB column, new graph entity, new FE page/component, or new enum transition ⇒ `L3-feature`.

— **Step 3 — L3-feature routing exit.** `L3-feature` classifications stop the workflow before Step 4. The skill prints a routing report (reason, affected entities from Stage 2, verbatim `/nacl-sa-feature "<description>"` command to run in a fresh session, disambiguation path back into `/nacl-tl-intake`) and exits. No files written, no graph nodes created, no code touched. The previous "create MINIMAL specification" path is removed; the "Bug in area with no docs at all" edge case is rewritten to route. An `--force-l3-spec-gap` escape hatch handles the rare genuine mis-classification.

— **Step 6M — migration verification sub-flow.** Whenever a fix adds or modifies a SQL migration: (1) pre-check the migrator manifest (drizzle `meta/_journal.json`, knex `knex_migrations`, prisma timestamped dir, custom fallback) — register the new file before running migrate; (2) run the declared migrate command — exit 0 is necessary but not sufficient; (3) post-check the DB with `SELECT COUNT(*) FROM <table> WHERE <pre-migration-condition>` — must return 0. Silent skips that previously produced `PASS` now produce `RUNNER_BROKEN`. The Step 8 report's "Changes applied" section gains a `Migration verification:` block recording manifest registration, the exact migrate command, and the post-check result. Trust the DB, not the exit code.

— **Step 6c — Path A anchor for brand-new files.** Brand-new files (not in the git tree before the fix) force `Path A` unconditionally. A file that did not exist could not have been imported by any test, so the import grep is meaningless for it. Closes the historical inversion where "no import found" was misread as "Path B (no test needed)" and shipped without a regression test.

— **Step 7.5 — data-flow survey.** The two-bullet impact check is replaced by five mandatory items the agent must answer in the Step 8 report: read paths (every `CONSUMES`/`REFERENCES` UC's reading code), write paths (every `PRODUCES`/`MUTATES`/`AFFECTS_ENTITY` UC's writing code), refresh / sync / cache / re-derivation (the question that catches "the migration fixed the DB but the next refresh will undo it"), snapshot vs source-of-truth (if a migration mutates rows, where is the source — code or user data?), and adjacent UCs / shared types. If any item 1–4 cannot be answered with a concrete file path and a stated verification, the fix status downgrades to `UNVERIFIED`. The agent does not silently call `PASS` while neighbours are unexamined.

Two cross-cutting principles thread through the release: the graph is a structural impact map (not a name index — Stage 2 traversal beats keyword search), and the fix skill is not a feature factory (`/nacl-sa-feature` is). No invocation syntax changed; the only new flag is `--force-l3-spec-gap`.
