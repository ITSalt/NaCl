NaCl 2.16.0 — intake-self-diagnosis

Релиз о том, что фреймворк перестаёт перекладывать собственную домашнюю работу на пользователя. Автономный оркестратор, который строит гипотезы и просит человека их проверить, — не автономный. Теперь «граф проекта не разрешил атом» — не повод задать вопрос, а повод провести расследование.

Что внутри:

— **Самодиагностика intake (PROBE).** Для каждого атома, который граф не разрешил, `nacl-tl-intake` формулирует 2–3 фальсифицируемые гипотезы («механизм есть в коде, но неверно обрабатывает запись» vs «механизма нет — это фича») и проверяет их ограниченными read-only пробами: grep/чтение кода, максимум один read-only запрос к БД, git log; бюджет 6 вызовов на атом, никаких записей. Новый уровень доказательств `CODE` — «проверено по реальному коду/БД».

— **Оцифрованная уверенность.** «Насколько я уверен» — теперь число, которое оператор может аудировать и настраивать, а не ощущение. Score — детерминированный lookup по паттерну вердиктов (модель не выдумывает число); пороги в `config.yaml → intake.*`. ≥0.9 — атом роутится как граф-подтверждённый; 0.7–0.9 — авто-роут по лидирующей гипотезе с трекаемой альтернативой; ниже 0.7 — и только там — вопрос, и он обязан нести диагноз: что проверено, вердикты по гипотезам, к чему склоняюсь, какой единственный факт мешает. Голых «баг или фича?» во фреймворке больше нет.

— **Переклассификация вместо падения.** BUG-атом, который `/nacl-tl-fix` доказательно признал фичей (кодового пути не существует), больше не валит goal-ран. FEATURE_SMALL самоисцеляется в том же ране; FEATURE_HEAVY уходит в новый терминальный стейт `unsupported` — ран продолжается. Ограниченная цена ошибки — то, что оправдывает авто-роутинг с 0.7 вместо вопроса.

— **Валидационный контур догнал L8–L13.** Пострелизный аудит 2.15 подтвердил: сами валидаторы полны (52 проверки L8–L13). Но alias `validate` в goal-контракте объявлял успех по l1–l7 (автономный цикл «не заметил» бы CRITICAL в новых слоях), финализация не видела ни один из 12 новых типов узлов, а `nacl-sa-full` вообще не вызывал новые продьюсеры. Исправлено: контракт требует все тринадцать уровней; readiness стал adoption-aware («не принят» ≠ 0%); у sa-full появилась опциональная Фаза 6b (машины → слайсы → ошибки → устойчивость, строго по зависимостям); tl-plan встраивает машину состояний экрана в task-fe.md. Публичная документация методологии теперь несёт полный каталог L1–L13 (EN+RU).

Всё аддитивно. Новые Cypher-запросы прогнаны вживую на реальных графах; intake-харнес — пять детерминированных тестов на смерженном дереве.
