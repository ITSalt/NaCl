# Packaged workflow migration record

The Codex plugin keeps the current 60-workflow methodology as internal
resources and exposes ten bounded conductor skills for discovery. This avoids
the initial skill-description budget problem without changing workflow names,
closed statuses, confirmation gates, or the Claude Code source distribution.

The package is a mechanical copy/adaptation boundary. Source workflow changes
must be deliberately ported and revalidated; the installed cache never reads a
checkout as a fallback. Graph lifecycle, project registry, secrets, and agent
profile delivery remain later-wave contracts.
