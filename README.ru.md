# NaCl

**NaCl** (Na + Cl) -- набор из **59 агентских скиллов** для Claude Code и Codex, охватывающих полный цикл разработки ПО -- от бизнес-анализа и системной спецификации до TDD-разработки, код-ревью, QA и релиза. Артефакты бизнес- и системного анализа хранятся в графе Neo4j, поэтому каждое требование можно запросить, отследить и оно никогда не потеряется в толще Markdown.

## Как это работает

```
 Клиентский документ
        |
        v
 /nacl-init "Мой проект"        Инициализация репозитория
        |
        v
 /nacl-ba-full                     Бизнес-анализ --> Neo4j граф
        |                           (процессы, сущности, роли,
        |                            правила, глоссарий)
        v
 /nacl-sa-full                     Системная спецификация --> Neo4j граф
        |                           (архитектура, домен, UC, UI,
        |                            роли, валидация)
        v
 /nacl-tl-conductor                Планирование + оркестрация
        |
        +-- /nacl-tl-plan          Задачи из графа (волны, BE+FE пары)
        +-- /nacl-tl-dev-be              Бэкенд TDD
        +-- /nacl-tl-dev-fe              Фронтенд TDD
        +-- /nacl-tl-review              Код-ревью
        +-- /nacl-tl-qa                  E2E тестирование (Playwright)
        +-- /nacl-tl-ship                Коммит, пуш, PR
        |
        v
 /nacl-tl-deliver --> /nacl-tl-release        Staging --> Production
```

## Что внутри

Все скиллы следуют конвенции `nacl-{слой}-{действие}`: **BA** = Business Analysis, **SA** = System Analysis, **TL** = TeamLead.

| Категория | Префикс | Кол-во | Описание |
|-----------|---------|--------|----------|
| **Бизнес-анализ** | `nacl-ba-*` | 14 | Процессы, сущности, роли, правила, глоссарий, импорт документов, валидация |
| **Системный анализ** | `nacl-sa-*` | 10 | Архитектура, доменная модель, Use Cases, UI, роли, валидация |
| **TeamLead** | `nacl-tl-*` | 26 | Разработка: TDD (BE/FE), ревью, QA, деплой, релиз, хотфикс, диагностика |
| **Утилиты** | `nacl-*` | 6 | `nacl-core`, `nacl-render`, `nacl-publish`, `nacl-init`, `nacl-goal`, `nacl-postmortem` |
| **Миграция** | `nacl-migrate-*` | 3 | Детерминированная миграция Markdown → Neo4j граф с паттерном адаптеров. |
| | | **59** | |

### Язык скиллов

BA и SA скиллы по умолчанию работают **на русском** (аналитические артефакты предназначены для бизнес-стейкхолдеров), TL-скиллы -- **на английском** (код и коммиты пишутся на английском). Фактический язык вывода определяется не языком, на котором написан сам SKILL.md (большинство BA/SA SKILL.md сейчас написаны на английском), а порядком разрешения: явный флаг `--lang=en`/`--lang=ru` → `project.lang` в `config.yaml` → дефолт слоя. См. `nacl-core/lang-directive.md`.

### Совместимость сред

NaCl поставляет пакеты скиллов для Claude Code и Codex.

- Claude Code использует корневые папки `nacl-*`, установленные в
  `~/.claude/skills/`.
- Codex использует адаптированный пакет `skills-for-codex/`, установленный в
  `$HOME/.agents/skills/`.

См. [Установка скиллов](docs/setup/install-skills.ru.md) для команд под нужную
среду и ОС.

> Скиллы **не работают** на claude.ai/code (веб-версия), потому что она запускается в песочнице без доступа к локальной файловой системе. Claude Code **Desktop** (нативное приложение) полностью поддерживается через оба канала ниже; не поддерживаются только облачные/веб-сессии.

### Выберите канал

| Вы используете | Устанавливаете как | Команда |
|---|---|---|
| Claude Code CLI | Симлинки на скиллы (этот checkout репозитория) | `sh scripts/install-claude-code-skills.sh` |
| Claude Code Desktop | Плагин | `/plugin marketplace add ITSalt/NaCl`, затем `/plugin install nacl@nacl` |
| Codex | `skills-for-codex/` (без изменений) | `sh skills-for-codex/scripts/install-user-symlinks.sh` |

На одной машине выбирайте один канал Claude Code -- не устанавливайте одновременно
симлинки скиллов и плагин. Технически оба варианта рабочие, но они дублируют одни и те
же скиллы под разными именами, и SessionStart-хук плагина предупредит, если обнаружит
уже установленный второй канал. См. [Настройка графа](docs/setup/graph-setup.ru.md) —
там описана специфика графовой инфраструктуры для Desktop (автоопределение Docker
Desktop, автозапуск sidecar, закреплённый бинарник `neo4j-mcp`).

## Быстрый старт

**1. Клонируйте репозиторий**

```bash
git clone https://github.com/ITSalt/NaCl.git
```

**2. Установите скиллы для вашей агентской среды**

Выберите Claude Code или Codex и следуйте инструкции
[docs/setup/install-skills.ru.md](docs/setup/install-skills.ru.md).

**3. Инициализируйте проект**

Откройте вашу агентскую среду в директории проекта, затем выполните:

```
/nacl-init "Название проекта"
```

`/nacl-init` сам разворачивает граф Neo4j: поднимает Docker Compose стек внутри вашего
проекта, дожидается его готовности, загружает схему и записывает `.mcp.json`. Запускать
`docker compose up` из этого checkout вручную не нужно.

**4. Запустите бизнес-анализ**

```bash
> /nacl-ba-full
```

Далее следуйте пайплайну: `nacl-sa-full` --> `nacl-tl-conductor` --> `nacl-tl-deliver` --> `nacl-tl-release`.

Подробная инструкция: [docs/quickstart.ru.md](docs/quickstart.ru.md)

## Архитектура агентов

NaCl направляет каждый скилл к одному из 6 когнитивных агентов, подбирая сложность задачи к нужной модели Claude:

| Агент | Модель | Скиллов | Зона ответственности |
|---|---|---|---|
| **strategist** | Opus | 12 | Архитектура, валидация, глубокое ревью |
| **analyst** | Sonnet | 13 | Доменное моделирование, структурированный контент, парсинг для миграции |
| **developer** | Sonnet | 6 | TDD-генерация кода, исправление багов |
| **verifier** | Sonnet | 5 | Тестирование, верификация, проверка контрактов |
| **operator** | Sonnet | 8 | Git-операции, CI/CD, публикация, оркестрация миграции |
| **scout** | Haiku | 6 | Быстрые запросы, статусные проверки |

7-й агент, **diagnostician** (Opus), не получает скилл верхнего уровня — он вызывается саб-агентом для диагностико-спековой фазы `/nacl-tl-fix` (Шаги 1–5), так что рассуждения багфикс-скилла идут на Opus, а генерация кода остаётся на Sonnet.

Определения агентов находятся в `.claude/agents/`. Подробная аргументация выбора модели: [docs/agents.md](docs/agents.md).

## Миграция из Markdown

Уже есть проект с Markdown-документацией по BA/SA? Пайплайн миграции конвертирует её в граф:

```
/nacl-migrate [project_path]
```

Миграция использует детерминированный парсинг на Python (без LLM) с паттерном адаптеров для разных форматов Markdown. Подробнее: [docs/migration.md](docs/migration.md).

## Передача графа (Handover)

Передача графа Neo4j проекта между машинами — это одноразовый зашифрованный экспорт/импорт. Используется при передаче проекта другому разработчику или переезде на новую рабочую станцию. Подробнее: [docs/HANDOVER.ru.md](docs/HANDOVER.ru.md).

## NaCl Analyst Tool

**NaCl Analyst Tool** -- локальное веб-приложение, которое заменяет «голый» Docker-контейнер Excalidraw. Оно показывает дерево всех бордов из `graph-infra/boards/`, отображает статус синхронизации каждого борда с графом Neo4j и предоставляет кнопки **Regenerate**, **Sync** и **Analyze**, запускающие NaCl-скиллы через `itsalt-pinch`. Старые контейнеры `excalidraw` и `excalidraw-room` из `graph-infra/` больше не нужны. Запуск: `cd analyst-tool && npm install && npm run dev` -- интерфейс откроется на `http://localhost:3582`.

Инструмент поддерживает работу с несколькими проектами через единый процесс-демон: инициализируйте каждый проект командой `/nacl-init "Название проекта"` из его директории (это регистрирует проект в `~/.nacl/projects.json`), а затем переключайтесь между ними через выпадающий список в шапке приложения без перезапуска сервера. Полное руководство и раздел «Multi-project setup»: [docs/analyst-tool.md](docs/analyst-tool.md#multi-project-setup).

## Структура проекта

```
NaCl/
  .claude/agents/     7 определений когнитивных агентов (strategist, diagnostician, analyst, developer, ...)
  nacl-ba-*/          14 скиллов бизнес-анализа
  nacl-sa-*/          10 скиллов системной спецификации
  nacl-tl-*/          26 скиллов жизненного цикла разработки
  nacl-migrate-*/      3 скилла миграции Markdown → граф
  nacl-core/          общие Cypher-хелперы и утилиты графа
  nacl-render/        рендеринг в Markdown и Mermaid (рендеринг Excalidraw перенесён в analyst-tool)
  nacl-publish/       публикация в Docmost
  nacl-init/          инициализация проекта
  nacl-goal/          обёртка над командой /goal
  nacl-postmortem/    инструментарий пост-мортемов скиллов
  nacl-tl-core/       общие шаблоны и справочники TL
  graph-infra/        Docker-инфраструктура Neo4j (шаблон копируется в каждый проект командой /nacl-init)
  plugin/             коммиченный артефакт плагина Claude Code Desktop (собирается scripts/build-plugin.mjs)
  .claude-plugin/     манифест маркетплейса для канала Desktop-плагина
  skills-for-codex/   адаптированный пакет скиллов для канала Codex
  analyst-tool/       Локальный веб-инструмент для работы с Excalidraw-бордами (запускается отдельно: `cd analyst-tool && npm install && npm run dev`)
  docs/               документация
```

Каждый скилл -- это директория с файлом `SKILL.md`, который Claude Code загружает при вызове слэш-команды. SKILL.md содержит роль агента, входные/выходные артефакты, алгоритм работы, критерии качества и примеры.

## Участие в проекте

Мы рады вкладу в проект. Ознакомьтесь с [руководством для контрибьюторов](docs/contributing.ru.md) перед отправкой pull request.

Основные правила:
- Каждый скилл автономен и содержит полный SKILL.md
- BA/SA скиллы по умолчанию выводят результат на русском, TL -- на английском (реальный язык вывода определяется `--lang` / `config.yaml`, а не языком, на котором написан сам SKILL.md)
- Все данные хранятся в графе, файлы -- только для кеша
